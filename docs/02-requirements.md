# 요구사항 정의서

## 1. 문서 목적

이 문서는 금융기관조회서 대사 자동화 웹/API 서비스의 기능 요구사항, 비기능 요구사항, 검증 규칙, 제외 범위를 정의한다.

본 프로젝트의 1차 목표는 금융기관조회서 정리 Excel과 회사 제시 명세서 Excel을 업로드받아, 대사 식별자 기준으로 데이터를 매칭하고 금액 및 주요 항목의 일치 여부를 검증하는 것이다.

대사 식별자는 금융상품이나 조회 대상 항목을 식별하는 번호를 의미한다. 예를 들어 예금의 계좌번호, 차입금의 대출번호, 보험의 증권번호, 보증의 보증번호, 금융상품의 계약번호 등이 이에 해당한다.

## 2. 사용자

### 주요 사용자

- 감사 업무 담당자
- 회계 감사 보조자
- 금융기관조회서 대사 업무 수행자

### 사용자의 주요 목적

- 조회서 금액과 회사 명세서 금액의 일치 여부를 빠르게 확인한다.
- 수작업 XLOOKUP 반복 작업을 줄인다.
- 누락 항목, 금액 불일치, 중복 항목을 자동으로 탐지한다.
- 검증 결과를 Excel 파일로 받아 감사 조서에 활용한다.

## 3. 1차 MVP 기능 요구사항

### 3.1 간단한 웹 화면 제공

시스템은 사용자가 브라우저로 접속할 수 있는 간단한 웹 화면을 제공해야 한다.

웹 화면에는 다음 요소가 포함된다.

- 조회서 Excel 파일 선택 버튼
- 회사 명세서 Excel 파일 선택 버튼
- 대사 실행 버튼
- 대사 결과 요약 표시 영역
- 결과 Excel 다운로드 버튼

1차 MVP에서는 디자인보다 기능 동작을 우선하며, React 또는 Vue 같은 별도 프론트엔드 프레임워크는 사용하지 않는다.

### 3.2 조회서 Excel 업로드

사용자는 금융기관조회서 내용을 정리한 Excel 파일을 업로드할 수 있어야 한다.

조회서 Excel에는 최소한 다음 컬럼이 포함되어야 한다.

| 컬럼명 | 설명 | 필수 여부 |
| --- | --- | --- |
| item_identifier | 대사 식별자. 예: 계좌번호, 대출번호, 증권번호, 계약번호, 보증번호 | 필수 |
| confirmation_amount | 조회서상 금액 | 필수 |
| item_type | 금융 항목 유형. 예: DEPOSIT, LOAN, INSURANCE, GUARANTEE | 선택 |
| institution_name | 금융기관명 | 선택 |
| currency | 통화 | 선택 |
| base_date | 기준일 | 선택 |

### 3.3 회사 명세서 Excel 업로드

사용자는 회사가 제시한 금융기관 관련 명세서 Excel 파일을 업로드할 수 있어야 한다.

회사 명세서 Excel에는 최소한 다음 컬럼이 포함되어야 한다.

| 컬럼명 | 설명 | 필수 여부 |
| --- | --- | --- |
| item_identifier | 대사 식별자. 예: 계좌번호, 대출번호, 증권번호, 계약번호, 보증번호 | 필수 |
| book_amount | 회사 명세서상 금액 | 필수 |
| item_type | 금융 항목 유형. 예: DEPOSIT, LOAN, INSURANCE, GUARANTEE | 선택 |
| institution_name | 금융기관명 | 선택 |
| currency | 통화 | 선택 |
| base_date | 기준일 | 선택 |

### 3.4 대사 식별자 기준 매칭

시스템은 조회서 Excel의 `item_identifier`를 기준으로 회사 명세서 Excel에서 동일한 대사 식별자를 찾아야 한다.

이는 Excel의 XLOOKUP과 유사한 역할을 한다.

처리 기준은 다음과 같다.

- `item_identifier` 앞뒤 공백은 제거한다.
- 비교 시 하이픈, 공백 등 구분자 제거를 고려할 수 있다.
- `item_identifier`가 비어 있으면 검증 실패로 처리한다.
- 동일한 `item_identifier`가 회사 명세서에 없으면 누락으로 처리한다.
- 동일한 `item_identifier`가 여러 건 존재하면 중복으로 처리한다.

1차 MVP에서는 `item_identifier` 단독 매칭을 기본으로 한다.  
향후에는 다음과 같은 복합 키 매칭을 고려한다.

```text
institution_name + item_type + item_identifier + currency
